CSS选择器

基本选择器

  • Type（类型）选择器 element
  • Class（类）选择器 .classname
  • ID选择器 #idname
  • 通用选择器 *
  • 属性选择器
  1. [attribute]，如：[target] 选择带有 target 属性所有元素。
  2. [attribute=value]，如：[target=_blank] 选择 target=”_blank” 的所有元素。
  3. [attribute~=value]，如：[title~=flower] 选择 title 属性包含单词 “flower” 的所有元素。
  4. [attribute|=value]，如：[lang|=en] 选择 lang 属性值以 “en” 开头的所有元素。
  5. 带正则表达式一类的，如[attribute^=value]，[attribute$=value]，[attribute*=value]

组合选择器

  • 紧邻兄弟选择器 A+B
  • 一般兄弟选择器 A~B
  • 子选择器 A>B
  • 后代选择器 A B

伪类选择器

  • 常见伪类的：
    :link、:visited、:active、:hover、:firstline、:first-child、:before、:after、:nth-child(n)
  • 常见的伪元素：::after、::before、::first-letter、::first-line、::selection、::backdrop

CSS中的小知识：

  • 可继承的属性：font-size, font-family, color
  • 不可继承的样式：border, padding, margin, width, height

CSS优先级

优先级就是分配给指定的CSS声明的一个权重。

CSS优先级：内联(style="") > 内联样式表(<style>) | 外链样式表(<link>) > 浏览器缺省

注：内联样式表和外链样式表取决于定义的位置顺序。

选择器优先级：ID选择器 > 类选择器 | 属性选择器 | 伪类选择器 > 元素选择器

!important: 当在一个样式声明中使用一个!important 规则时，此声明将覆盖任何其他声明。

指定不同选择器的权重，计算权重值。如：

  1. 内联样式表的权值最高 1000；

  2. ID 选择器的权值为 100

  3. Class 类选择器的权值为 10

  4. HTML 标签选择器的权值为 1

CSS盒模型

  • IE 怪异盒子模型：
    总宽度= width + margin(左右)（即width已经包含了padding和border值）
  • W3C标准盒子模型：
    总宽度= width + margin(左右) + padding(左右) + border(左右)

CSS设置盒模型：

  • 怪异盒模型：box-sizing: border-box;
  • 标准盒模型：box-sizing: content-box;

CSS浮动

概念

div是块级元素，在页面中独占一行，自上而下排列，也就是标准流。

浮动可以理解为让某个div元素脱离标准流，漂浮在标准流之上，和标准流不是一个层次。

float用于设置标签的居左浮动和居右浮动，浮动后的元素不属于html文档流。

无论行内元素还是块元素，被设置浮动之后，display属性值都变为block。

影响

  • 影响同级元素

多个同级元素，前者设置浮动，则脱离标准流，”浮到上面”，后者会自动填补
空白，造成叠加现象。

  • 父级元素塌陷

父级嵌套子集盒子，父级未设置宽高，子集浮动，则父级失去”被撑开的自身”物理空间。

清除

  1. 给父元素定义高度
  2. 给父元素加上 overflow：hidden 或者 overflow：auto　(BFC)
  3. 给父元素加上display: table
  4. 在最后一个子元素上加一个div，使用clear：both
  5. 父元素定义伪类：:after { content:""; display:block; clear:both; }

BFC

定义

BFC(Block formatting context)直译为”块级格式化上下文”。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

布局规则

  • 内部的Box会在垂直方向，一个接一个地放置。
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠。每个元素的margin box的左边，与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算。

参考网站：https://www.w3ctech.com/topic/865

生成BFC

  1. 根元素
  2. 浮动元素（float属性不为none）
  3. 绝对定位元素 （ position属性为 absolute 或 fixed）
  4. display为inline-block, table-cell, table-caption, flex, inline-flex
  5. overflow 值不为 visible 的块元素

块格式化上下文对于定位与清除浮动很重要。

CSS隐藏元素

  • 使用 opacity 透明度属性 opacity: 0;
  • 设置 Visibility 可见性 visibility: hidden;
  • 使用 display 显示性 display: none;
  • 使用固定定位移出屏幕 position: absolute; left: -9999px;
  • 将一个元素设置为缩放无限小 transform: scale(0);
  • CSS3 属性，将一个元素的模糊度设置为0 filter: blur(0);

CSS动画

  1. transition（过渡）
  2. animation（动画）
  3. transform（变形）

部分问题

  1. display:inline-block 什么时候会显示间隙？
  • 相邻的 inline-block 元素之间有换行或空格分隔的情况下会产生间距
  • 非 inline-block 水平元素设置为 inline-block 也会有水平间距
  • 可以借助 vertical-align:top; 消除垂直间隙
  • 可以在父级加 font-size：0; 在子元素里设置需要的字体大小，消除垂直间隙
  • 把 li 标签写到同一行可以消除垂直间隙，但代码可读性差
  1. link 与 @import 的区别
  • link属于XHTML标签，而@import是CSS提供的;
  • 页面被加载的时，link会同时被加载，而@import引用的CSS会等到页面被加载完再加载;
  • import只在IE5以上才能识别，而link是XHTML标签，无兼容问题;
  • link方式的样式的权重 高于@import的权重。
